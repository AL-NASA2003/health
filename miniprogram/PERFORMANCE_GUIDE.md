# 微信开发者工具性能优化指南

## 🎯 常见问题解决

### 1. 小程序加载慢

#### 快速优化步骤：

1. **关闭不必要的功能**：
   - 关闭 "自动预览"
   - 关闭 "文件监听"（如果不需要热更新
   - 关闭 "网络请求日志"（可选
   - 关闭 "AppData 实时预览"（可选

2. **调整项目设置**：
   - 设置 → 项目设置 → 关闭 "ES6 转 ES5"（如果已在代码中处理
   - 关闭 "压缩代码"（开发阶段可以关闭，提升编译速度
   - 开启 "不校验合法域名"（开发阶段

3. **调试器优化**：
   - 调试器设置 → 关闭 "保留登录态"
   - 只开启需要的面板（不要同时开太多

### 2. 数据同步问题

#### 已实现的优化：
✅ 使用事件总线进行实时数据同步
✅ 添加数据变更标记
✅ 减少并发请求，分批加载
✅ 优化请求超时时间（8s → 5s）
✅ 减少重试次数（2次 → 1次）

#### 验证数据同步：
1. 在 "我的记录" 页面添加一条饮食记录
2. 回到首页，应该立即看到统计更新
3. 查看控制台，应该有相关的日志输出

### 3. 网络请求超时

#### 检查后端服务：
```bash
# 确保后端服务正在运行
# 打开浏览器访问 http://localhost:5000/api/user/info
```

#### 如果后端启动慢：
- 确保数据库已连接
- 检查是否有爬虫任务在运行（可以暂时禁用
- 查看后端日志是否有错误

## 🚀 快速测试步骤

1. **启动后端**：
   ```bash
   python run.py
   ```

2. **打开微信开发者工具**：
   - 导入项目
   - 关闭不必要的功能

3. **测试数据同步**：
   - 在首页查看初始数据
   - 进入"我的记录"添加记录
   - 返回首页，查看数据是否立即更新
   - 查看控制台日志确认

## 💡 其他建议

1. **真机调试更快**：微信开发者工具模拟器相对较慢，建议经常使用真机调试
2. **定期清理缓存**：工具 → 清除缓存 → 清除全部缓存
3. **简化测试数据**：开发阶段使用少量数据，避免数据库过大
4. **关注控制台日志**：新增了很多调试日志，方便排查问题

## 🔍 常见错误排查

### 错误：网络请求失败
1. 检查后端是否正常运行
2. 检查端口5000是否被占用
3. 查看后端控制台日志

### 错误：数据同步不更新
1. 查看首页控制台是否有事件监听日志
2. 检查"我的记录"页面是否发送了数据变更事件
3. 确认事件名称是否匹配
