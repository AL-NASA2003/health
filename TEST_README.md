# 健康饮食管理系统 - 系统测试脚本

## 简介

这是一个符合毕业设计要求的综合系统测试脚本，用于验证健康饮食管理系统的各项功能。

## 测试内容

测试脚本包含以下13个测试模块，共18个测试用例：

1. **系统初始化** - 测试系统基础功能
   - 根路径访问
   - 数据库连接

2. **用户管理** - 测试用户相关功能
   - 用户登录
   - 获取用户信息

3. **食谱功能** - 测试食谱相关功能
   - 获取食谱列表
   - 食谱搜索

4. **食材管理** - 测试食材相关功能
   - 获取食材列表

5. **饮食记录** - 测试饮食记录功能
   - 获取饮食记录

6. **运动记录** - 测试运动记录功能
   - 获取运动记录

7. **饮水记录** - 测试饮水记录功能
   - 获取饮水记录

8. **论坛功能** - 测试论坛相关功能
   - 获取帖子列表
   - 发布帖子

9. **手账功能** - 测试手账相关功能
   - 获取手账列表
   - 添加手账

10. **个性化推荐** - 测试个性化推荐功能
    - 个性化食谱推荐

11. **热点美食** - 测试热点美食功能
    - 获取热点美食列表

12. **健康指标** - 测试健康指标功能
    - 获取健康记录

13. **API文档** - 测试API文档访问
    - API文档访问

## 使用方法

### 运行测试

```bash
python system_test.py
```

### 测试环境

确保以下条件满足：
- Python 3.7+
- Flask应用已正确配置
- 数据库已正确初始化
- 所有依赖已安装（requirements.txt）

## 测试结果

测试完成后会输出详细的测试报告，包括：
- 总测试数
- 通过测试数
- 失败测试数
- 通过率
- 总耗时
- 每个测试的详细结果和耗时
- 总体评价

## 示例输出

```
总测试数: 18
✓ 通过: 18
通过率: 100.0%
总耗时: 0.64秒

总体评价：系统运行良好，所有核心功能正常！
```

## 注意事项

1. 测试使用测试账号 `test`/`123456` 进行登录
2. 测试会创建临时数据（帖子、手账等），但不会影响现有数据
3. 建议在测试环境中运行，避免影响生产数据
4. 确保后端服务正常运行

## 毕业设计集成

此测试脚本完全符合毕业设计要求：
- 覆盖系统所有核心功能模块
- 提供清晰的测试报告
- 验证系统各项功能正常工作
- 可作为毕业设计答辩演示内容
